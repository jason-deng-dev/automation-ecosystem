---
name: Explain before implementing new concepts
description: For new concepts or APIs Jason hasn't used before, explain the approach first before writing code
type: feedback
---

When a task involves something new to Jason (new Node.js APIs, new patterns, etc.), explain what it is and how it works before implementing it. Don't just write the code silently.

**Why:** Jason is learning, not just shipping. He wants to understand every part of the system. Moving too fast skips the learning moment.

**How to apply:** Before implementing anything touching a concept Jason hasn't seen yet (e.g. `fs/promises`, `writeFile`, cron, Claude API), briefly explain what it does and why, then let Jason have a go or confirm he wants you to implement it.
